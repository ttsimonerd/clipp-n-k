Build a production-minded self-hosted SaaS web app called **clipp'n'k**.

## What this app is

A web platform for gamers to upload, trim/crop, store, and share game clips and short videos. It should feel like a polished alternative to catbox.moe, but specifically optimized for sharing clips in Discord and similar chat apps.

## Core product requirements

* Users must log in with **Discord OAuth only**.
* After login, the app must verify that the user is a member of a **configured Discord server**.
* Only users in that server can access the app.
* There must be a personal dashboard for each user.
* Users can upload clips/videos.
* Users can trim/crop videos inside the website.
* Users can generate short shareable URLs for clips.
* Shared clip pages must look good in Discord previews and open cleanly in browser.
* The app must be designed to be **easy to self-host**.

## Very important admin requirement

* The Discord server ID used for membership checking must be **changeable from an admin settings page**.
* The host/admin should not need to edit code to change the guild/server ID.
* The admin UI should also be able to manage basic settings like:

  * upload limits
  * allowed file types
  * branding title/logo/colors
  * default visibility for uploaded clips
  * maximum clip duration or file size
  * whether new uploads are public or private by default

## Deployment requirement

Make the app easy to run on:

* **Coolify**
* **Vercel**
* Docker-based deployments
* other common self-hosting setups

Prefer a portable architecture with:

* environment variables
* clear setup steps
* minimal platform-specific assumptions
* support for containerized deployment
* a setup that works both locally and in self-hosted environments
